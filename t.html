Tor

在我们通常访问的网络外,还有两个特殊的网络,一个是暗网一个是深网,它们不被人们所知,但是依然存在.此文只对使用tor访问的网络做出说明.

名词

上网: 本文中所有的“上网”指的是上到暗网.
验证: 本文中所有的“验证”指的是签名验证.

安全总则

  1. 软件的验证

    任何来自于网络的信息和软件,都需要验证其所有者,否则即为不可信软件.
    处于网络工具中的软件 必须在安装之前,经过验证,否则不得使用!!
    上传到网络(网盘、邮件等)后,又从上传位置下载下来的软件,在第一次运行之前,需要重新验证.
    将已验证过的软件发送给伙伴B时,伙伴B下载本地后,在第一次运行前,需要重新验证.

    总之,无论软件的来源是什么,在下载到主机后,安装之前(无需安装的,在第一次运动前),必须通过验证.

  2. 软件的使用建议

    尽量不要使用大陆出品的软件.
    上网的主机,除了必需的网络工具之外不要安装其它软件,包括 QQ 360安全, WeChat等,专机专用! 建议使用背后无商业公司的linux系统,如debian,kali等,如果使用的是windows系统,请打开Windows Defender.

    如果没有干净系统的主机,可以使用 virtaulbox 或者 vmware 安装一个干净的系统用于上网,专机专用.

Tor Browser

有关Tor的前世今生,请自行通过网络补习,在此不做赘述. 只说明一个如何正确使用此软件.

  1. 下载

使用软件的第一步是下载和安装软件,由于Tor的特殊性,其下载和安装需要较多的工作以保证其安全性.所以以下的步骤不能省略!!

在Tor的官网下载应用, 请验证网站的正确性,包括ssl证书. 下载完成后,切记请先不要安装. 在下载到本地时要同时下载软件的验证文件,以备验证使用.

  1. 验证文件

验证的时机: 安装之前(无需安装的,在第一次运行前)

将下载的 TorBrowser-11.5.4-osx64_en-US.dmg.asc 文件及 TorBrowser-11.5.4-osx64_en-US.dmg 文件放入到同一个目录中,如~/tor目录 验证 Tor 是否在被修改,以下以MacOS 为例:

# 1. 安装gpg,此软件为验证Tor所使用的工具

brew install gpg


# 2. 验证所下载的 Tor

# 在MacOS 下使用 空格 + command  调出命令行 
# 进入到下载的目录,假设目录为 ~/tor
cd ~/tor

# 查看目录下的文件,可以使用如下的命令,也可以使用mac OS Finder,检查是否有一对文件
# TorBrowser-11.5.4-osx64_en-US.dmg.asc 文件
# 以及 TorBrowser-11.5.4-osx64_en-US.dmg 文件
ls 

# 执行验证,输入如下命令,回车
gpg --verify TorBrowser-11.5.4-osx64_en-US.dmg.asc

# 注意查看输出内容,其中包含
# gpg: Good signature from "Tor Browser Developers (signing key) <torbrowser@torproject.org>" [unknown],
# 表示验证成功

# 可能会有: gpg: WARNING: This key is not certified with a trusted signature!,
# 这句话的意思是: 不能保证和相应的人是挂钩的. 这个可以忽略.

验证过程中可能出现的问题:

  • gpg: Can't check signature: No public key

会出现如下提示:

gpg: assuming signed data in 'TorBrowser-11.5.4-osx64_en-US.dmg'
gpg: Signature made Sat 27 Aug 2022 02:34:37 AM CST
gpg:                using RSA key E53D989A9E2D47BF
gpg: Can't check signature: No public key

这个问题是由于没有公钥引起的,从出错信息中可以看到,没有公钥的 key 是: E53D989A9E2D47BF 记下它,然后可以通过如下命令解决

# 这个参数 E53D989A9E2D47BF 来自于出错提示信息
gpg --search-keys E53D989A9E2D47BF

# 出现:
# Keys 1-1 of 1 for "E53D989A9E2D47BF".  Enter number(s), N)ext, or Q)uit > 
# 输入数字1, 然后回车, 此时 公钥会被下载并导入到本地.

有公钥后,再次执行验证,就不会有错误提示了.

即执行:

cd ~/tor

# 查看目录下的文件,可以使用如下的命令,
# 也可以使用mac OS Finder,检查是否有一对文件

#  TorBrowser-11.5.4-osx64_en-US.dmg 文件
#  以及TorBrowser-11.5.4-osx64_en-US.dmg.asc
ls 

# 执行验证,输入如下命令,回车
gpg --verify TorBrowser-11.5.4-osx64_en-US.dmg.asc

在输出中就可以找到: gpg: Good signature from "Tor Browser Developers (signing key) <torbrowser@torproject.org>" [unknown]

至此,可以证明,我们所下载的 Tor 是安全的.

  1. 安装 Tor

直接在图形界面中双击,按照默认设置,“下一步” 即可.

  1. 设置代理服务器

在 Tor 中设置代理服务器, Tor 网络有很多网桥等中继服务,此处的代理服务不是指这部分内容,而是指先将tor代理到远程的服务器,再进行网络访问.

设置tor的代理服务可以通过修改Tor Network Setttings来完成.

此处要依据您购买的代理服务器信息,正确的输入

  • 代理类型
  • 代理的地址
  • 代理的端口

如果你的代理有用户名和密码,请一并输入.

使用大陆的代理服务器会导致Tor完全不能上网,所以请选择其它地区的代理服务.

网站

做完以上步骤,您就可以使用Tor来上网了,上网之前,有以下几点提示.

所有使用 tor 访问的暗网都以.onion 作为域名的结尾,为了隐藏,前面是一串“乱码”.

导航网站有各种 tor 站点的地址,这些地址有一些被标记为认证,但你时刻要记得,在这个网络里,没有任何保证网站上所说的内容是真实的,也没有任何保证你受害后可以有挽救的措施.

和我们通常接触的网站一样,网络中的站点也会跑路,原因可能各不相同,但是要有这个基本的风险意识.

在网络中进行的交易完全依靠对方的“自觉”,没有人能保证在这里面不会被骗,所以做好风险识别.

总结

总的来说,使用tor上网并不太难,就4个步骤: 下载,验证,安装,设置代理,然后就可以上网了. 如果您对安全要求不高,例如只是想体验一下,那完全可以跳过第二步的验证和第四步的设置代理(前提是不在国内),直接下载、安装、上网了,但是需要说明的是,一旦安装过未经验证的tor的操作系统,就不能作为工作机使用了.

工作环境下的tor安装完成后,这台机器(或者虚拟机), 就专机专用了,尽量不要安装其它的软件,如需安装,则在需安装之前进行文件的验证,验证通过后方可安装.

对安装完成后的工作环境,要做好保护,比如登录系统需要输入密码,以便以后继续使用. 如果是虚拟机,还可以将其拷贝到U盘上,这样可以在其它地方使用.
